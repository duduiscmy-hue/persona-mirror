# 人格镜像馆 · Persona Mirror

> **"每个人不是被分类，而是在8个维度上有0-100的连续得分，形成一个独特的'人格DNA'。"**

🔗 **在线访问**: https://persona-mirror.pages.dev

---

## 一、创意构思

### 1.1 灵感来源

传统人格测试（如MBTI）将人分为16种离散类型，但真实的人格是**连续的、流动的、情境依赖的**。本项目受"光谱"概念启发，拒绝标签化，采用**8维连续光谱模型**，让每个人都能得到一个几乎独一无二的"人格DNA"。

### 1.2 核心概念：数字分身

用户不仅是测试的参与者，更是"数字分身"的造物主。通过：
- **人格探测**：生成8维DNA编码
- **分身训练**：录入情境反应、语言风格、情绪模式
- **镜像对话**：访客可与主人的数字分身实时对话

### 1.3 目标用户

- 想深度了解自己的大学生
- 对心理学/人格分析感兴趣的人群
- 想通过"数字分身"与远方朋友保持连接的人

## 二、功能模块（6个交互模块）

| 模块 | 交互内容 | 技术亮点 |
|------|----------|----------|
| **1. 人格探测** | 22道电影场景式情境选择题，8维计分，生成雷达图+DNA编码 | 情境化题目设计（非直白提问），E选项开放输入+句式分析 |
| **2. 分身训练** | 自由对话分析（5轮开放对话，前端实时语言风格分析）+ 10个情境反应库 + 4种情绪光谱 | 纯前端规则引擎分析回复长度、标点情绪、高频词、人称比例、否定词、时间取向、情绪词 |
| **3. 镜像对话** | 访客输入名字后与主人数字分身聊天 | 基于规则的回复生成：情境识别→情绪分析→DNA调参→模板匹配→语言档案微调 |
| **4. 人格匹配** | 输入两人DNA，计算契合度+组合解析+相处建议 | 12种TOP2组合名称+15种相遇解析+动态生成建议 |
| **5. 塔罗彩蛋** | 22张大阿卡纳抽牌，CSS 3D翻转动画 | 纯CSS动画，随机牌阵 |
| **6. 档案馆** | 查看历史测试、训练档案、匹配记录、对话记录，支持JSON导出 | LocalStorage持久化+数据可视化 |

### 2.1 进阶功能（超出课程要求）

- **云端同步**：接入Supabase，实现跨设备数据同步
- **用户认证**：昵称+密码注册/登录，云端保存档案
- **主人权限**：主人账号可查看全站私密档案
- **自动归档**：测试完成后自动弹窗选择"私密/公开"
- **云端失败回退**：网络异常时自动回退本地登录，保证可用性

### 3.2 技术栈

| 层级 | 技术 | 说明 |

| 版本控制 | Git + GitHub | 代码托管与版本管理 |

### 3.3 核心算法

**8维计分逻辑**：
- 初始各维度50分
- 每题选项影响特定维度 ±5分
- E选项（开放输入）根据句式结构分析分配分数
- 封顶0-100

**规则引擎回复生成**：
```
访客输入 → 关键词情境识别 → 情绪色彩分析 → 
人格DNA调参（长度/直接度/温暖度/幽默度/追问倾向） → 
匹配情境库模板 → 语言档案微调 → 输出回复
```

**匹配解析系统**：
- 12种TOP2组合名称（如"理性织网者""社交探险家"）
- 15种经典相遇解析（如"双网交织""蓝图与故事"）
- 动态生成：共享维度共鸣 / 异质吸引 / 平行线交汇 / 星尘相遇

**云端安全策略（RLS）**：
- 普通用户：只能SELECT自己的user_archives记录
- 主人用户：通过`isMaster=true`标记，可查看全站所有档案
- 邮箱验证关闭：使用合成邮箱`@persona.local`，简化注册流程

## 四、设计亮点

### 4.1 题目设计：电影场景式

每道题都是一个小故事场景，选项没有明显性格标签：

> **第1题：咖啡馆的雨天**
> 周五下午，你窝在沙发里刷手机。窗外雨声渐密，朋友突然发来一张照片——街角一家亮着暖黄灯光的咖啡馆...你会：
> - A. 立刻截图搜地址，发给三个朋友："明天下午去？我请客"
> - B. 长按保存图片，新建相册命名为"雨天灵感"，觉得这种氛围很适合写点什么
> - C. 回复"一起去？"然后切回短视频，那张图淹没在信息流里
> - D. 没回复，但脑子里开始编故事：这家咖啡馆的老板娘其实是个时间旅行者...
> - E. 以上都不是，我会：________

### 4.2 结果展示：人格图腾

不显示无意义的十六进制编码，而是：
- **人格图腾名称**：根据TOP2维度生成（如"理性织网者"）
- **程度条+通俗解释**：`决策风格 █████████░ 82% — 习惯用逻辑分析做决定 ⭐`
- **交叉解读**：基于TOP3和BOTTOM3维度的个性化分析
- **基于你的回答**：引用用户具体回答进行解读

### 4.3 情绪光谱：优先级排序

不是单选，而是给每种情绪的多个反应方式**打1-5星**，更真实反映人的复杂反应模式。

### 4.4 云端同步与本地回退

- 网络正常时：数据自动同步到Supabase云端，跨设备登录同一账号即可看到全部档案
- 网络异常/云端故障时：自动回退到LocalStorage本地模式，保证网站可用性
- 主人权限：特殊账号可查看全站所有用户的私密档案

## 五、使用方式

### 5.1 本地运行

```bash
# 克隆仓库
git clone https://github.com/duduiscmy-hue/persona-mirror.git
cd persona-mirror

# 方式一：直接双击 index.html（部分功能受限）
# 方式二：启动本地服务器（推荐）
python3 -m http.server 8000
# 然后访问 http://localhost:8000
```

### 5.2 在线访问

直接访问：https://persona-mirror.pages.dev

### 5.3 使用流程

1. **注册/登录**：输入昵称+密码（自动创建账号）
2. **人格探测**：完成22题测试，查看8维雷达图和人格图腾
3. **分身训练**：
   - 完成5轮自由对话，让系统分析你的语言风格
   - 填写10个情境反应库
   - 设置情绪光谱优先级
4. **镜像对话**：输入访客名字，与主人的数字分身聊天
5. **人格匹配**：输入两组DNA，查看契合度和相遇解析
6. **档案馆**：查看历史记录，导出JSON数据

## 七、项目反思

### 7.1 遇到的挑战

1. **开放输入的计分精度**：用户输入混合多种倾向（如"先问位置，下次随缘，可能自己去也可能和朋友"），简单关键词匹配导致分数互相抵消。解决方案：增加句式结构分析（"如果...就...""要么...要么...""随缘/看情况"等条件从句识别）。

2. **代码完整性**：AI生成的代码有时不完整（如`startQuiz`函数缺失、文件复制时丢失），导致按钮无法点击。解决方案：强制刷新浏览器、检查文件大小、使用`git pull`同步最新代码。

3. **浏览器安全策略**：`file://`协议下Supabase等API无法正常工作。解决方案：使用VS Code Live Server启动本地HTTP服务器。

4. **云端同步复杂性**：接入Supabase后遇到`supabase`变量重复声明、RLS策略配置、邮箱验证关闭、主人权限设置等问题。解决方案：逐一排查错误信息，修改代码逻辑，在Supabase控制台配置安全策略。

5. **缓存问题**：部署后页面未更新。解决方案：强制刷新（Cmd+Shift+R）。

### 7.2 未来优化方向

1. **接入真AI**：当前规则引擎有局限，未来可接入LLM API提升对话自然度
2. **社交图谱**：增加用户间的匹配网络可视化
3. **动态题目**：根据用户历史回答动态调整后续题目
4. **多语言支持**：增加英文版本
5. **移动端App**：将网页封装为PWA或原生App
